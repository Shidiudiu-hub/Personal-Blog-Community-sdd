# 个人博客社区系统 - 功能需求文档

## 一、项目概述

### 1.1 项目定位
个人博客社区系统是一个前后端分离的PC端Web应用，旨在为用户提供文章发布、阅读、互动交流的社区平台。

### 1.2 技术架构
- **前端技术栈**：Vue 3 + Element Plus + Vite + Pinia + Vue Router + Axios
- **后端技术栈**：Spring Boot + JPA/Hibernate + MyBatis + JWT认证
- **数据库**：H2（开发环境，可切换MySQL）
- **文件存储**：OSS对象存储（已配置）

### 1.3 已有功能
- ✅ 用户注册/登录
- ✅ 用户管理（CRUD）
- ✅ JWT认证机制
- ✅ 基础权限管理
- ✅ 前端管理后台模板

---

## 二、核心功能模块

### 2.1 文章管理模块

#### 2.1.1 功能描述
用户可以在平台上发布、编辑、管理自己的文章，支持Markdown格式编辑和预览。

#### 2.1.2 功能清单
1. **文章发布**
   - Markdown编辑器（使用@toast-ui/editor）
   - 实时预览功能
   - 文章标题、摘要设置
   - 分类和标签选择
   - 封面图片上传
   - 文章状态设置（草稿/发布）

2. **文章列表**
   - 个人文章列表（我的文章）
   - 全部文章列表（社区文章）
   - 分类筛选
   - 标签筛选
   - 搜索功能（标题、内容）
   - 排序功能（最新、最热、最多点赞）

3. **文章详情**
   - 文章内容展示（Markdown渲染）
   - 作者信息展示
   - 发布时间、阅读量、点赞数、收藏数
   - 分类和标签展示
   - 相关文章推荐

4. **文章编辑**
   - 编辑已发布的文章
   - 编辑草稿
   - 删除文章（软删除）

#### 2.1.3 数据模型设计
**Article（文章表）**
- article_id: 文章ID（主键）
- user_id: 作者ID（外键）
- title: 文章标题
- content: 文章内容（Markdown格式）
- summary: 文章摘要
- cover_image: 封面图片URL
- category_id: 分类ID（外键）
- status: 状态（0-草稿，1-已发布，2-已删除）
- view_count: 阅读量
- like_count: 点赞数
- collect_count: 收藏数
- comment_count: 评论数
- create_time: 创建时间
- update_time: 更新时间
- deleted: 逻辑删除标记

#### 2.1.4 接口设计
- `POST /api/article/publish` - 发布文章
- `POST /api/article/update` - 更新文章
- `GET /api/article/list` - 获取文章列表（分页）
- `GET /api/article/detail` - 获取文章详情
- `GET /api/article/my-list` - 获取我的文章列表
- `POST /api/article/delete` - 删除文章
- `GET /api/article/search` - 搜索文章

- create_time: 创建时间
- update_time: 更新时间

**ArticleTag（文章标签关联表）**

#### 2.2.4 接口设计

### 2.3 评论回复模块

#### 2.3.1 功能描述
用户可以对文章进行评论，支持多级回复（评论的评论），形成评论树结构。

#### 2.3.2 功能清单
1. **评论功能**
   - 发表评论
   - 编辑自己的评论
   - 删除自己的评论
   - 评论列表展示（树形结构）
   - 评论分页加载

2. **回复功能**
   - 回复评论
   - 回复回复（多级嵌套）
   - @用户提醒
   - 回复列表展示

3. **评论互动**
   - 评论点赞
   - 评论数量统计

#### 2.3.3 数据模型设计
**Comment（评论表）**
- comment_id: 评论ID（主键）
- article_id: 文章ID（外键）
- user_id: 评论用户ID（外键）
- parent_id: 父评论ID（0表示顶级评论）
- content: 评论内容
- like_count: 点赞数
- reply_count: 回复数
- status: 状态（0-正常，1-已删除）
- create_time: 创建时间
- update_time: 更新时间
- deleted: 逻辑删除标记

**CommentLike（评论点赞表）**
- id: 主键
- comment_id: 评论ID（外键）
- user_id: 用户ID（外键）
- create_time: 创建时间

#### 2.3.4 接口设计
- `POST /api/comment/add` - 发表评论
- `POST /api/comment/reply` - 回复评论
- `GET /api/comment/list` - 获取评论列表（树形结构）
- `POST /api/comment/update` - 更新评论
- `POST /api/comment/delete` - 删除评论
- `POST /api/comment/like` - 点赞/取消点赞评论
- `GET /api/comment/count` - 获取文章评论数

## 四、业务流程

### 4.1 文章发布流程
1. 用户登录后进入"发布文章"页面
2. 填写文章标题
3. 使用Markdown编辑器编写内容
4. 选择分类和标签
5. 上传封面图片（可选）
6. 填写文章摘要（可选）
7. 点击"保存草稿"或"发布"
8. 系统保存文章并返回文章详情页

### 4.2 评论流程
1. 用户在文章详情页查看评论
2. 在评论输入框输入评论内容
3. 点击"发表评论"
4. 系统保存评论并刷新评论列表
5. 其他用户可以看到评论并回复

### 4.3 点赞收藏流程
1. 用户在文章详情页或列表页看到点赞/收藏按钮
2. 点击按钮，系统判断当前状态
3. 如果未点赞/收藏，则执行点赞/收藏操作
4. 如果已点赞/收藏，则取消操作
5. 更新按钮状态和数量显示

### 4.4 关注流程
1. 用户在用户主页或用户卡片看到"关注"按钮
2. 点击"关注"按钮
3. 系统创建关注关系
4. 更新关注状态和数量
5. 在"我的关注"列表中可以看到该用户

## 五、权限设计

### 5.1 用户角色
- **普通用户**：可以发布文章、评论、点赞、收藏、关注
- **管理员**：拥有所有权限，可以管理分类、标签、用户

### 5.2 权限控制
- 未登录用户：只能浏览文章，不能发布、评论、点赞、收藏
- 已登录用户：可以发布文章、评论、点赞、收藏、关注
- 文章作者：可以编辑、删除自己的文章
- 评论作者：可以编辑、删除自己的评论
- 管理员：可以管理所有内容

## 十一、API接口规范

### 11.1 统一响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```

### 11.2 分页响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "list": [],
    "total": 100,
    "page": 1,
    "size": 20
  }
}
```

### 11.3 错误码定义
- 200: 成功
- 400: 参数错误
- 401: 未登录
- 403: 无权限
- 404: 资源不存在
- 500: 服务器错误
